System Requirements

    Operating Systems:

    • Linux 2.4.20 kernel
    • Linux 2.6.22 kernel
    • VxWorks Tornado 2.1(END)

    Hardware:

    • BCM94705LGR, BCM94704agr, BCM94704mpcb  -or-
    • Base Reference Design with a BCM94321 MiniPCI card -or-
    • Base Reference Design with a BCM94322 MiniPCI card -or-
    • Base Reference Design with a BCM94312 SDIO module -or-
    • Vendor Equivalent

What's New in Release 5.70.30?

  • Based on wl driver version 5.60.
  • 5357: update on gainctrl
  • Add auto channel support for 4313.
  • 5357: Increasing the DMA buffers from 256 TO 320, to overcome ET rxoverflow issue.

  • Add support for 4313 for VxWorks router

What's New in Release 5.70.29?

  • Based on wl driver version 5.60.91
  • 4331 sample driver candidate with all calibration and tx power control
  • Fixnig the wrong noise level reporting because of wrong index calculations.
  • Increase the NRXBUFPOST to overcome frequent rxoflow issue seen with 5357.
  • 4313: Fix not restoring ntssi_delay, nvbat_delay, ntemp_delay and intg_log2 values for tempsense based tx pwr ctrl

What's New in Release 5.70.28?

  • Based on wl driver version 5.60.88
  • 5357: Update Gainctrl, complete Noise Mitigation, update ACI mitigation
  • CTF: Changes to avoid possible pkt leak, specially for the slow packets

What's New in Release 5.70.27?

  • Based on wl driver version 5.60.85
  • 5357B0 external PA Tx tuning and gain table update
  • 5357/43236/6362: Spur avoidance enabled

What's New in Release 5.70.26?

  • Based on wl driver version 5.60.84
  • Ensure that the AFE Bandgap never get shut down un-intentionally during Tempsense
  • Fix EMF is blocking uPnP traffic
  • Add BCM4748 chip support
  • 5357 PLL settings for spur avoidance modes.
  • For extLNA based designs, make init gain backoff parameterizable based on nvram field elna2g/5g.
  • When re-assembling the TKIP encrypted packet fragments, don't adjust the body_len for WEP IV when LINUX_CRYPTO is defined, because the kernel tkip_decrypt() has already stripped off the WEP IV from the 802.11 header.

What's New in Release 5.70.25?

  • Based on wl driver version 5.60.79
  • Drop Tx gain on 5357B0
  • AP ACS: provide an option to treat 11B APs equally.
  • Fixing issues related to power save enable for non-brcm stas.

What's New in Release 5.70.24?

  • Based on wl driver version 5.60.78
  • Fix for the tput dip issue seen on 5357 NREPA

What's New in Release 5.70.23?

  • Based on wl driver version 5.60.76
  • 43236/5357: Update for Tempsense + Rx Gain Control Updates
  • MFGC: Non aggregated traffic may be sent during the first second of a new IBSS.
  • WiFi WPA2: Test 4.2.3.6 and 4.2.3.7.2 fails due to Use_Protection (B1) bit not being set. (Changes to enable the BCN_POMISC bit for AP.)

What's New in Release 5.70.22?

  • Based on wl driver version 5.60.75
  • PHY changes for 6362
  • Add spur-avoidance support for HTPHY

What's New in Release 5.70.21?

  • Based on wl driver version 5.60.73
  • First 4331 driver with 3 streams rate working.
  • We will not generate extra VLAN TAG (required for 53115 WAR for Vista router test case) if TOS of IP packet has non-zero priority.

  • Do not prevent selecting 40MHz channel when detecting interference (ACS related).
  • Initialize the fallback rate set properly for 4331.
  • 4331: need to expand rssi calculation for 3x3
  • Enable 4331 as supported chip
  • Enable 20/40 Coexistence by default.

  • Initial support for Radio On/Off Duty Cycle for power save.
